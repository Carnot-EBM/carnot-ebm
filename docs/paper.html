Carnot: Energy-Based Verification and Repair of Large Language Model Output

Ian Blenke
Independent Researcher
April 2026  ·  github.com/Carnot-EBM/carnot-ebm  ·  carnot-ebm.org  ·  Markdown  ·  Models
Abstract

Large Language Models (LLMs) generate fluent but unverifiable text through autoregressive token prediction, with no mechanism to enforce logical consistency or factual accuracy. We present Carnot, an open-source dual-language (Rust + Python/JAX) framework that applies Energy-Based Models (EBMs) to verify, score, and repair LLM output. Through 22 systematic experiments on a 596M-parameter model (Qwen3-0.6B) and cross-model analysis with Qwen3.5-0.8B, we establish a practical hierarchy of energy signals for LLM output improvement. Our composite verification architecture — combining per-token log-probabilities with structural test execution — achieves +10% accuracy on factual QA and lifts code generation from 0% to 30% correct, with no model fine-tuning required. For constraint satisfaction, gradient-based repair improves LLM accuracy from 60% to 80% on random 3-SAT. Per-token activation EBMs trained via NCE achieve 84.5% test accuracy in distinguishing correct from hallucinated tokens. Conversely, activation-space directions with 0.945 AUROC detection power produce zero causal effect when injected during generation. We also find that instruction tuning compresses the hallucination signal: base models (84.5%) have larger activation gaps than instruction-tuned models (67.2%), meaning the models most in need of hallucination detection are the hardest to detect on. We distill 8 empirically-grounded principles and release the complete framework under Apache 2.0.

1. Introduction

1.1 The Verification Gap

Autoregressive language models generate text by sequentially predicting the most probable next token conditioned on the preceding context. This architecture produces remarkably fluent output but provides no mechanism to verify that the generated sequence satisfies logical constraints, is factually accurate, or is internally consistent. When an incorrect token is generated early in a sequence, the error propagates irrecoverably — the model conditions on its own mistakes. This fundamental limitation manifests as hallucination: the production of plausible but incorrect output.

The verification gap is structural, not parametric. Scaling model size, training data, or compute does not eliminate it because autoregressive generation lacks a feedback mechanism to check completed output against global constraints.

1.2 Energy-Based Models as Verifiers

Energy-Based Models (EBMs) assign a scalar energy $E(x) \in \mathbb{R}$ to a complete configuration $x$, where low energy corresponds to valid configurations. This enables:

  1. Holistic evaluation. The energy function evaluates the entire output simultaneously.
  2. Gradient-based repair. When $E(x) > 0$, gradient descent fixes violated components: $x_{t+1} = x_t - \eta \nabla_x E_{\text{violated}}(x_t)$.
  3. Verifiable certification. $E(x) = 0$ constitutes a mathematical proof that all constraints are satisfied.
  4. Composability. Multiple constraints compose additively: $E_{\text{total}}(x) = \sum_i w_i E_i(x)$.

1.3 Contributions

  1. A composite verification architecture that improves QA accuracy by +10% and code generation from 0% to 30%, with no model modification.
  2. Gradient repair of LLM constraint satisfaction, improving 3-SAT accuracy from 60% to 80%.
  3. Per-token activation EBMs achieving 84.5% test accuracy — the first activation-based approach to generalize beyond calibration data.
  4. A definitive negative result on activation steering: 0.945 AUROC detection → 0% generation effect.
  5. Cross-model activation analysis showing instruction tuning compresses the hallucination signal (84.5% → 67.2%).
  6. Eight empirical principles for EBM-LLM integration, distilled from 22 experiments.
  7. Carnot, an open-source dual-language framework under Apache 2.0. Pre-trained models at huggingface.co/Carnot-EBM.

2. Related Work

Energy-Based Transformers. Hoover et al. (2025) demonstrate that EBTs achieve 35% faster scaling than Transformer++ and 29% improvement via iterative refinement. This validates energy-based inference at transformer scale.

Autoregressive Models as EBMs. Zhao et al. (2025) establish a formal bijection between autoregressive models and EBMs, proving that every LLM's per-token log-probabilities constitute a valid energy function. Our Experiment 13 empirically confirms this.

Semantic Energy. Farquhar et al. (2025) detect hallucination via negative log-likelihood. Our logprob rejection sampling independently converges on this approach (+10% improvement).

Concept Vectors. Anthropic (2025) demonstrates that concept-specific activation vectors are causally effective for steering, while generic mean-difference directions are not. Our Experiments 15–16 and 20 corroborate this: generic hallucination directions show 0.945 AUROC separation but zero causal effect.

Existing Frameworks. EB-JEPA (Meta FAIR, PyTorch) focuses on self-supervised world modeling; THRML (Extropic, JAX) targets thermodynamic hardware; TorchEBM (PyTorch) provides general-purpose EBM tooling. Carnot is the first to combine dual-language implementation (Rust + JAX), specification-driven development, and an automated self-improvement loop.

3. Framework Architecture

3.1 Dual-Language Design

Carnot implements the same abstractions in both Rust and Python/JAX, connected via PyO3:

  • Rust (7 crates): core traits, three model tiers, samplers, training, PyO3 bridge
  • Python/JAX (8 modules): mirrors Rust with JAX-native implementations using Flax and Optax
  • Cross-language serialization: safetensors format for interop

3.3 Constraint Verification

Constraints are encoded as differentiable energy terms. Each constraint provides energy(x) → scalar (0 = satisfied) and grad_energy(x) → ∇x for repair. A ComposedEnergy aggregates constraints with per-constraint verification reports and violated-only gradients.

Implemented domains: Boolean satisfiability (product relaxation), graph coloring (pairwise repulsion), Python code (execution-based), and property-based testing (random invariants).

3.4 Composite Scoring

For each candidate c ∈ {c₁, ..., cₙ}:
    logprob_score(c) = mean per-token log-probability
    structural_score(c) = fraction of test cases failed
    E(c) = -w₁ · logprob_score(c) + w₂ · penalty · structural_score(c)

Select c* = argmin E(c)
If E(c*) > threshold: iterate with failure feedback to LLM

4. Experiments

All experiments use Qwen3-0.6B (596M parameters) unless noted. We chose a small model deliberately: if EBM verification works on a weak model, it will work on stronger ones.

4.1 SAT Gradient Repair

Setup. 20 random 3-SAT instances at the phase transition (12 variables, 40 clauses). The LLM generates variable assignments; gradient descent on the continuous relaxation repairs violations.

✅

Result: LLM accuracy 60% → 80% (+20%). Of 8 incorrect instances: 4 fully repaired, 2 partially improved, 2 not repaired. Multi-start repair (N=10) fixes one additional instance.

4.2 Hallucination Detection via Activations

Setup. 25 factual questions. Extract per-layer activations from generated tokens (not prompts). Hallucination direction = mean difference between correct and incorrect activations.

🔍

Result: Detection accuracy: 80%. AUROC: 0.945. Energy gap: +9.3. The signal is real — but as Section 4.6 shows, it does not translate to causal control.

4.3 Logprob Rejection Sampling

Setup. 20 questions, 5 candidates each ($T = 0.7$). Select by highest mean per-token log-probability.

✅

Result: Greedy 45% → logprob-selected 55% (+10%). No calibration, no training, no external model needed. The model's own logprobs ARE the energy function.

4.4 Composite Energy for Code

Setup. 10 coding tasks, 5 candidates each. Composite energy: $E(c) = -w_1 \cdot \bar{\ell}(c) + w_2 \cdot p \cdot f(c)$.

✅

Result: Greedy 0% → composite-selected 30%. Structural tests dominate for code; logprobs dominate for QA. The composite is never worse than either alone.

4.6 In-Generation Activation Steering

Setup. Subtract hallucination direction from hidden states during generation. 6 configurations: upper/middle/all layers × $\alpha \in \{0.1, 0.5, 1.0, 2.0, 5.0\}$.

⚠️

Result: Zero effect across ALL configurations. Zero fixes, zero regressions. The 0.945 AUROC direction produces exactly 0% change when used for intervention. Statistical separation ≠ causal influence.

Motivation. Experiments 9–12 fail because mean-pooling destroys per-token signal. Training on individual token activations yields 288× more examples (26,800 vs 93).

4.8 Cross-Model Activation Analysis

Motivation. If per-token activation EBMs work on base model activations, do they transfer to instruction-tuned models? Instruction-tuned models are the ones actually deployed, so practical hallucination detection must work on them.

Setup (Experiments 21–22). Collected 52,296 per-token activations from Qwen3.5-0.8B (instruction-tuned with thinking capabilities) on factual QA (300 pairs) + TruthfulQA (817 adversarial questions). Trained per-token Gibbs EBMs under three conditions.

The instruction-tuning paradox: RLHF training produces models that sound confident even when wrong — internal activation patterns become more uniform regardless of factual accuracy. The models most in need of hallucination detection are precisely the ones where activation-based detection is hardest. Mixing activations from different models degrades performance (70.5%), confirming different models occupy different representation spaces.

5. Principles for EBM-LLM Integration

Principle 1

Simpler is better in small-data regimes. Linear projections outperform nonlinear models with <100 examples. The Gibbs EBM achieved 94% calibration but 35% test — textbook overfitting.

Principle 2

Token-level features dominate sequence-level. Mean-pooling destroys the hallucination signal. Per-token logprobs (+10%) and per-token activations (84.5%) succeed where sequence-level fails.

Principle 3

The model's own confidence is the best energy signal. No external EBM outperformed log-probabilities for candidate selection. The model is already an energy-based model (ARM↔EBM bijection).

Principle 4

Overfitting is the primary failure mode. Every trained approach overfits when $n_{\text{examples}} \ll d_{\text{features}}$. The fix: more data (Experiment 19) or no training (Experiment 13).

Principle 5

Extract features from generated tokens, not prompts. Prompt activations are identical across candidates. The signal is only in the generated tokens.

Principle 6

Different energy signals dominate different domains. Logprobs for QA. Structural tests for code. The composite handles both and is never worse than either.

Principle 7

Statistical separation ≠ causal influence. A direction at 0.945 AUROC does not steer generation (0% effect). Effective steering requires concept-specific vectors, not generic contrastive directions.

Principle 8

Instruction tuning compresses the hallucination signal. Base models (84.5%) have larger activation gaps between correct and wrong answers than instruction-tuned models (67.2%). RLHF makes models sound confident even when wrong — the models most in need of hallucination detection are the hardest to detect on.

6. Autonomous Self-Improvement Loop

Carnot implements an automated research loop (LLM proposes hypotheses, energy function evaluates):

  1. Propose. An agent generates candidate EBM improvements.
  2. Sandbox. Candidates execute in isolation (process-level dev, Docker+gVisor production).
  3. Evaluate. Three-gate check: energy improvement, execution time, memory.
  4. Learn. Trace2Skill extracts structured lessons into a skill directory.
  5. Repeat. Circuit breaker halts after N consecutive failures.

In a 50-iteration run with Claude 3.5 Sonnet, the loop achieved near-optimal energy (DoubleWell: 0.0001, Rosenbrock: 0.0092) before circuit breaker at iteration 18. The energy function serves as the objective judge — no human evaluation needed.

7. Discussion

7.1 What Works Today

The composite verification architecture is immediately deployable: no model modification, no fine-tuning, no calibration. The improvement is consistent: +10% QA, +30% absolute for code.

7.2 The Activation Ceiling and the Instruction-Tuning Paradox

Per-token activation EBMs achieve 84.5% on base model activations but plateau regardless of architecture. Cross-model analysis reveals a deeper challenge: instruction-tuned models yield only 67.2% detection accuracy — a 17.3pp drop. RLHF alignment improves average output quality but simultaneously makes remaining hallucinations harder to detect via activations. Practical deployment will require model-specific calibration.

7.3 The Steering Paradox

The complete disconnect between detection (0.945 AUROC) and steering (0% effect) is our most striking finding. Any approach using contrastive activation analysis for steering must demonstrate causal efficacy, not just separability.

7.4 Toward Hardware Acceleration

The Ising tier's quadratic energy maps directly to physical Boltzmann machines (Extropic TSU), opening a path to hardware-accelerated constraint satisfaction.

8. Limitations

  1. Model scale. All experiments use Qwen3-0.6B (596M). Results may differ on larger models.
  2. Task diversity. QA evaluation uses 20–93 questions; token-level experiments use 26,800 tokens from limited question diversity.
  3. Test case dependency. Code verification requires existing test cases.
  4. Activation ceiling. We have not identified what limits per-token EBM accuracy at ~84.5%.
  5. No fine-tuning comparison. We compare against unmodified LLM output, not RLHF/DPO baselines.

9. Conclusion

We demonstrate that Energy-Based Models provide a practical, training-free mechanism for improving LLM output. The most effective approach is surprisingly simple: use the model's own logprobs for candidate selection, compose with structural tests for code, and iterate with feedback. This improves QA (+10%) and code (0% → 30%) with no model modification.

Learned activation EBMs show promise at the per-token level (84.5% on base models) but plateau below the simple logprob baseline's practical impact. Cross-model analysis across 22 experiments reveals that instruction tuning compresses the hallucination signal (67.2% for instruction-tuned vs. 84.5% for base), creating a paradox where the models most deployed in production are hardest to monitor.

Our most impactful negative result — 0.945 AUROC detection → 0% steering effect — should caution the growing body of work on activation engineering: demonstrate causal efficacy, not just separability.

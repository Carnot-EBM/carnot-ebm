Carnot: Energy-Based Verification and Repair of Large Language Model Output

Ian Blenke
Independent Researcher
Abstract

Large Language Models (LLMs) generate fluent but unverifiable text through autoregressive token prediction, with no mechanism to enforce global consistency. We present Carnot, a dual-language (Rust + JAX) framework that applies Energy-Based Models (EBMs) to verify and repair LLM output. Through 25 systematic experiments on Qwen3-0.6B and Qwen3.5-0.8B, we demonstrate a composite verification architecture that improves QA accuracy by +10% (n=20 questions) and lifts code generation from 0% to 30% (n=10 tasks, vs. unmodified baseline). We establish that activation-space signals achieving 0.945 AUROC detection fail to produce causal effects during generation steering, highlighting a fundamental gap between detection and control. See Section 8 (Limitations) for statistical caveats.

1. Introduction

The fundamental limitation of autoregressive generation is the verification gap: models lack a feedback mechanism to check completed output against global constraints. Carnot bridges this gap by treating the generated sequence as a configuration in an energy landscape.

By assigning a scalar energy $E(x)$ to the output $x$, we enable holistic evaluation, gradient-based repair, and verifiable certification. This paper establishes the empirical principles for effective EBM-LLM integration.

Introspection, Not Fine-Tuning

Carnot never modifies the target LLM's weights. The language model remains completely frozen throughout all experiments. Our approach works by introspecting the model's existing internal representations:

  • Logprob methods read the LLM's own per-token log-probabilities — energy the model already computes. Per the ARM↔EBM bijection, every autoregressive model is already an EBM.
  • Activation methods extract hidden state activations from a frozen forward pass, then train a small separate EBM classifier on those features via Noise Contrastive Estimation.
  • Structural verification executes generated output against domain constraints. No model weights involved.

When we say “EBM training,” we mean training the small classifier on features from a frozen LLM — not gradient descent on the language model itself. This is closer to probing/introspection than to fine-tuning, RLHF, or DPO.

5. Core Principles

Principle 2 Token-level features dominate sequence-level. Mean-pooling activations destroys the hallucination signal. Per-token analysis is required for robust EBM training.
Principle 7 Statistical separation ≠ causal influence. A direction at 0.945 AUROC does not steer generation. Effective control requires concept-specific vectors, not generic contrastive directions.

6. Conclusion

Carnot demonstrates that Energy-Based Models provide a practical, training-free mechanism for improving LLM reliability. The Composite Verification architecture is immediately deployable and consistently superior to greedy decoding. Future work must address the instruction-tuning paradox, where RLHF alignment makes hallucinations harder to detect via internal activations.

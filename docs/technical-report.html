Energy-Based Verification for LLM Output

Technical Report on EBM-LLM Integration Strategy
Author: Ian Blenke  |  Date: 2026-04-06  |  License: Apache 2.0

Abstract

We present Carnot, an open-source framework that combines Energy-Based Models (EBMs) with Large Language Models (LLMs) to reduce hallucinations. Through 25 systematic experiments on Qwen3-0.6B and Qwen3.5-0.8B, we establish that the model's own log-probabilities are the most effective energy signal for candidate selection (+10% accuracy on 20 questions), while structural test execution dominates for code (0% → 30% on 10 tasks vs. unmodified baseline). Activation-space approaches show high detection power (0.945 AUROC) but zero causal effect when used for generation steering. This is alpha-stage research software; see the paper for statistical limitations.

1. Introduction

Large Language Models predict tokens based on local probability. Carnot introduces a holistic energy layer that evaluates entire outputs against global constraints. This enables holistic evaluation, gradient-based repair, and verifiable certification.

2. Framework Architecture

2.1 Dual-Language Implementation

Carnot provides identical abstractions in Rust (for performance) and Python/JAX (for research). Safetensors enable zero-copy model sharing.

2.2 Model Tiers

  • Ising: Quadratic energy, O(d²) complexity.
  • Gibbs: Multi-layer MLP verifier.
  • Boltzmann: Deep residual transformer-based verifier.

3. Key Results

3.1 SAT Gradient Repair

LLM accuracy on 3-SAT instances improved from 60% to 80% through gradient-based repair on continuous relaxations of the logical constraints.

3.2 Logprob Rejection Sampling

Selecting the candidate with the highest mean per-token log-probability improved QA accuracy by +10% (45% → 55%) without any training.

3.3 Hallucination Detection

Per-token activation EBMs achieve 84.5% test accuracy in distinguishing correct from hallucinated outputs. However, injecting these signals during generation produced 0% improvement, highlighting the gap between detection and causation.

4. Core Principles

1. Simpler is better. Linear projections outperform nonlinear models when training data is scarce (< 100 examples).
2. Token-level features dominate. Mean-pooling activations destroys the signal. Per-token analysis is required for robust verification.
3. Instruction tuning compresses signal. RLHF makes models sound confident even when wrong, making hallucination detection harder on instruction-tuned models (67% vs 84% on base).

5. Conclusion

The most effective deployment of EBMs today is the Composite Scorer: use logprobs for factual QA, structural tests for code, and iterate with feedback. Higher-level activation steering requires concept-specific vectors to overcome the causal gap.
